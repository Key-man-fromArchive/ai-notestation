## 1. 제품 정체성

### 우리는 무엇인가

LabNote AI는 **연구자의 운영 체제**다. 범용 노트 앱이 아니다.

문헌 탐색에서 논문 출판까지, 연구의 전 과정이 하나의 플랫폼 안에서 완결된다. AI가 각 단계에서 연구자의 인지 부담을 줄이고, 시스템 정의 카테고리가 팀의 지식을 구조화하며, Zotero 연동이 학술 생태계와 연결한다.

### 우리는 무엇이 아닌가

- Notion/Obsidian 같은 범용 PKM 도구가 아니다
- Benchling 같은 순수 전자실험노트(ELN)가 아니다
- Confluence 같은 범용 협업 위키가 아니다

### 차별화 요소

| 비교 대상 | 그들의 강점 | 우리의 차별점 |
|----------|-----------|-------------|
| **ELN** (Benchling, LabArchives) | 규정 준수, 실험 기록 | AI 네이티브 + 논문 워크플로우 통합 |
| **PKM** (Obsidian, SiYuan, Notion) | 범용 지식 관리, 커뮤니티 | 연구 프로세스 특화 블록 + 팀 관리 |
| **협업 도구** (Confluence, SharePoint) | 기업 문서 관리 | 연구 도메인 이해 + 실험/가설/결과 체계 |
| **학술 도구** (Zotero, Mendeley) | 서지 관리 | 실험-분석-논문까지 전 과정 통합 |

## 4. 비즈니스 모델 — 카테고리 기반 과금

### 핵심 원칙

**카테고리 = 기능 패키지 = 과금 단위**

시스템이 정의한 카테고리를 노트북에 지정하면, 해당 카테고리에 특화된 AI 프롬프트, 전용 블록 타입, 템플릿, 내보내기 옵션이 활성화된다. 카테고리는 자유 입력이 아닌 시스템 제공으로 파편화를 방지한다.

### 가격 구조

```
┌─ Free ──────────────────────────────────────────────┐
│  📝 자유 노트 (무제한)                                │
│  🔬 연구 노트 (기본 기능)                              │
│  📄 논문 리뷰 (Zotero 연동 제외)                       │
│  📋 회의록                                           │
│  멤버: 3명까지 | AI: 월 100회                         │
└─────────────────────────────────────────────────────┘

┌─ Pro ($15/user/month) ──────────────────────────────┐
│  Free 전체 +                                        │
│  📄 Zotero 완전 연동 (동기화 + Citation Picker)        │
│  🔬 프로토콜 라이브러리 + Immutable Lock               │
│  📊 프로젝트 관리 (간트 차트, 마일스톤, 예산)            │
│  📈 데이터 분석 (CSV, 차트, 통계 블록)                  │

│  전용 서버 배포 지원                                   │
```

## 5. 연구 파이프라인 — 7단계

LabNote AI의 핵심 가치. 연구의 전 과정이 하나의 플랫폼에서 완결된다.

```
📖 문헌 탐색  →  📝 문헌 리뷰  →  💡 가설  →  🧪 실험 설계  →  🔬 실험 수행  →  📊 분석  →  📄 논문
   Stage 1        Stage 2       Stage 3     Stage 4        Stage 5       Stage 6    Stage 7
```

### Stage 1: 문헌 탐색 & 수집

**기능**:
- Zotero 라이브러리 양방향 동기화 (Z-2)
- DOI 직접 입력 → 서지 정보 자동 생성
- URL 클립 → 메타데이터 추출
- 논문 상태 관리: 미읽음 → 읽는 중 → 읽음 → 리뷰 완료
- 읽기 목록 대시보드 + 진행률 표시

**AI 역할**:
- 논문 자동 요약 (abstract + 본문 기반)
- "이 논문과 관련된 내 기존 노트" 추천
- 읽기 우선순위 제안
- Zotero PDF 하이라이트 가져오기 → 색상별 의미 분류 (Z-3)

### Stage 2: 문헌 리뷰 매트릭스

**기능**:
- `/review-matrix` 블록 — 에디터 내 인라인 비교 테이블
- 열 정의: 방법론, 데이터셋, 결과, 한계점 등 (분야별 프리셋)
- 각 셀에서 원본 논문 노트로 [[링크]]
- CSV 내보내기

**AI 역할**:
- 논문을 행에 추가하면 AI가 각 열을 자동 채움
- 매트릭스 전체를 분석하여 **연구 갭(Research Gap)** 식별
- "아직 다루지 않은 방법론" 제안

### Stage 3: 가설 수립

**기능**:
- `/hypothesis` 블록 — 구조화된 가설 기록
  - 필드: 가설 ID, 제목, 진술, 근거 문헌, 관련 가설, 상태
  - 연결: 이 가설을 검증하는 실험들, 결과들
- 상태 흐름: 제안됨 → 검증 중 → 지지됨 / 기각됨 / 수정됨
- 가설에 연결된 실험 결과가 업데이트되면 상태 전환 제안

**AI 역할**:
- 팀 내 가설 중복/충돌 검사
- 문헌에서 이미 검증된 유사 가설 서피스
- 가설을 검증 가능한 실험 설계로 분해 제안

### Stage 4: 실험 설계 — 프로토콜 라이브러리

**기능**:
- 프로토콜 = 특수 유형 노트 (버전 관리 + 체크리스트 + 포크 가능)
- 재료 목록: 체크박스 + 단위(ml, μg, °C) + 수량
- "이 프로토콜로 실험 시작" → /experiment 블록 자동 생성
- 포크: 기존 프로토콜을 복사해서 수정 (원본 보존, 히스토리 추적)
- 팀 통계: 사용 횟수, 포크 수 → 가장 많이 쓰는 프로토콜 파악

**AI 역할**:
- 프로토콜 버전 간 차이점 요약
- 단계별 예상 소요시간 추정
- 재료 목록 누락 감지

### Stage 5: 실험 수행 & 기록

**기능**:
- `/experiment` 블록 — 구조화된 실험 기록
  - 필드: 날짜, 프로토콜 링크, 가설 링크, 실험자, 조건 테이블
  - 경과 기록: 타임스탬프 + 텍스트/음성/사진/데이터
  - 편차 기록: 예상과 다른 상황 ⚠️ 마킹
- Lab Mode (모바일/태블릿) 연동
- **Immutable Lock**: 서명 시 해시로 보존 → GLP/GMP 규정 준수
  - 서명 후 수정 불가, "추가 기록"만 가능

**AI 역할**:
- 사진 자동 OCR + Vision 분석
- 음성 메모 → 텍스트 변환 + 타임스탬프 삽입
- 편차 발생 시 잠재적 영향 경고

### Stage 6: 데이터 분석 & 결과

**기능**:
- `/result` 블록 — 실험/가설 링크, 결론, 통계
- 인라인 차트 (막대, 선, 산점도)
- 통계 수치 입력 (p-value, t-value, confidence interval)
- 여러 실험 결과 종합 메타 분석 뷰
- 가설 상태 자동 업데이트 제안

**AI 역할**:
- 편차가 결과에 미친 영향 분석
- 추가 실험 필요 여부 제안
- 통계적 유의성 해석
- 여러 실험 간 패턴 감지

### Stage 7: 논문 작성 & 출판

**기능**:
- Paper Draft Assistant — 프로젝트 내 블록 수집 → 섹션별 초안 생성
  - Introduction ← 문헌 리뷰 매트릭스
  - Methods ← 프로토콜 라이브러리
  - Results ← /result 블록들 + 차트
  - Discussion, Conclusion ← AI 종합
- Zotero Citation Picker (Z-1)로 인라인 인용 삽입
- 참고문헌 자동 생성 (Z-4) — 600+ CSL 스타일
- 내보내기: Word, LaTeX, PDF, Markdown
- 동료 리뷰: 실시간 협업 편집 + 코멘트 + 권한

**AI 역할**:
- 섹션별 초안 생성 (노트 기반, 인용 포함)
- 누락된 인용 감지
- 문체 일관성 검사
- 논문 구조 검증 (IMRAD 등)

## 6. Zotero 통합 — 5가지 핵심 기능

### 기술 기반

- **API**: Zotero Web API v3 (`https://api.zotero.org`)
- **인증**: `Zotero-API-Key` 헤더 (Settings에서 입력)
- **동기화**: Streaming API (실시간) + `since=<version>` (증분)
- **서지 포맷**: 600+ CSL 스타일
- **클라이언트**: Python (백엔드 동기화), TypeScript (프론트 Citation Picker)

### Z-1. Citation Picker (인용 삽입기)

에디터에서 `[@` 또는 `/cite` 입력 시 Zotero 라이브러리 검색 팝업.
- 퍼지 검색 (제목, 저자, 연도)
- 삽입 형식 선택: 인라인 `(Author, Year)`, 각주, 전체 서지
- CSL 스타일 선택: APA, IEEE, Chicago, Nature 등
- AI 추가: "이 맥락에서 인용할 만한 논문" 자동 제안

Obsidian 대비 차별점: 별도 마크다운 파일 생성 없이 **에디터 안에서 인라인 삽입**.

### Z-2. 자동 동기화 + 양방향 링크

- Zotero Collection ↔ LabNote 노트북 (논문 리뷰 카테고리) 매핑
- Zotero Item → LabNote 노트 자동 생성 (서지정보 + 메타데이터)
- Zotero PDF → LabNote 첨부파일 가져오기
- Zotero Tags ↔ LabNote 태그 양방향 동기화
- Streaming API로 실시간 감지

### Z-3. PDF 하이라이트 → 노트 변환

- Zotero PDF Reader의 하이라이트/주석을 API로 가져오기
- 색상별 의미 매핑: 🟡 배경지식 / 🟢 핵심 기여 / 🔴 한계 / 🔵 방법론
- 선택적 가져오기 또는 전체 노트 생성
- AI: 하이라이트 기반 논문 요약 자동 생성

### Z-4. 참고문헌 자동 생성

- 노트 내 모든 `[@citation]` 수집
- Zotero API `format=bib`으로 포맷된 참고문헌 목록 생성
- 누락 인용 자동 감지, 인용 순서 자동 정렬
- 내보내기(Word/LaTeX) 시 참고문헌 섹션 자동 생성

### Z-5. Zotero 그룹 → 팀 라이브러리

- Zotero Group ↔ LabNote 팀 공유 라이브러리 매핑
- 그룹 권한 → LabNote 권한 매핑
- 팀원의 논문 태그/메모 공유 → Team Insight에 "팀 문헌 트렌드" 반영

## 8. 인지과학 기반 설계 원칙

> 근거: [Deep Research 보고서 — PKMS 인지과학 분석](NotebookLM ainote-UIUX)

| **의미 형성 (부호화)** | 기존 지식과 연결 시 깊은 기억 | [[노트 링크]] + 백링크로 지식 간 연결 |

| **디지털 정원** | 지식은 시간 순서가 아닌 성장 과정 | 🌱→🌿→🌲 성숙도 시스템 |

## 11. 설계 원칙

1. **Research-Native** — 범용 노트가 아닌, 연구 프로세스에 최적화
2. **Category = Feature** — 카테고리가 기능을 결정하고, 기능이 가치를 만들고, 가치가 과금을 정당화
3. **AI is Ambient** — AI가 별도 페이지가 아니라 에디터 안에서 자연스럽게 동작
4. **Progressive Disclosure** — 기본은 심플, 파워 기능은 Cmd+K/슬래시/단축키
5. **Context Preservation** — 탭, 분할, 패널로 맥락 전환 최소화
6. **Keyboard First** — 모든 핵심 동작을 키보드로 완료 가능
7. **Bottom-Up Growth** — 대학 무료 → 기업 Pro → 기관 Enterprise
8. **Self-Hosted Trust** — 데이터가 사용자의 서버를 떠나지 않음
